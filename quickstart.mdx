This quickstart walks through a minimal Tandemn setup: an administrator starts the server, a user connects the CLI, and the user submits a batch inference job.

> You need a network-reachable control plane, AWS credentials, Redis, an S3 bucket, and Python 3.12+.

## What you will do

<Steps>
  <Step title="Clone the Tandemn server repository">
    Clone the Tandemn server repository and run setup.
  </Step>
  <Step title="Start the server">
    Configure `.env`, start Redis, and run the control plane.
  </Step>
  <Step title="Install the CLI">
    Install the `tandemn` Python package in your local environment.
  </Step>
  <Step title="Submit a job">
    Point the CLI at the server and run a first inference job.
  </Step>
</Steps>

## 1. Start the server

```bash
git clone --recurse-submodules https://github.com/Tandemn-Labs/Tandemn-server.git
cd Tandemn-server
bash setup.sh
```

Review `.env`, then start Redis.

```bash
docker run -d -p 6379:6379 redis
```

Start the control plane with the URL that EC2 replicas can reach.

```bash
python server.py --url https://your-public-url.example.com
```

> Keep the server URL handy. CLI users will set it as `TD_SERVER_URL`.

## 2. Install the CLI

Install the Tandemn CLI in a Python environment.

<CodeGroup>

```bash pip
pip install tandemn
```

```bash venv
python -m venv .venv
source .venv/bin/activate
pip install tandemn
```

</CodeGroup>

## 3. Connect to the server

Set the server URL and verify that the CLI can reach it.

```bash
export TD_SERVER_URL=<your-server-url>
tandemn check
```

If the check fails, see [Troubleshooting](/admin/troubleshooting).

## 4. Submit a job

Create or choose a JSONL prompt file, then submit it with a model and SLO.

```bash
tandemn plan Qwen/Qwen2.5-7B-Instruct prompts.jsonl --slo 4
tandemn deploy Qwen/Qwen2.5-7B-Instruct prompts.jsonl --slo 4
tandemn progress
```

You should see CLI output showing that the job was accepted by the server.
